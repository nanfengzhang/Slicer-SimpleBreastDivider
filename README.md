# SimpleBreastDivider (3D Slicer Extension)

这是一个专为计算医学与医学图像处理设计的 3D Slicer 扩展插件。本项目提供了一套完整的乳腺多模态影像高级分析工具集，致力于打通从原始医学影像到三维解剖结构量化分析的完整工作流。

## 🌟 核心功能 (Core Features)

1. **nnUNet 全自动深度学习分割**
   * 支持一键执行乳房整体区域与内部病灶靶点的全自动 3D 分割。
2. **空间约束过滤 (Spatial Constraint Filtering)**
   * 自动将病灶分割结果严格限制在解剖学乳腺掩膜范围内，极大消除假阳性区域。
3. **形态学解剖体积量化**
   * 提供精准的形态学边缘收缩（Erosion）算法，支持自定义收缩距离（默认 5.0mm），全自动计算解剖学上的乳腺体积上限值。
4. **真实腺体提取 (Glandular Tissue Extraction)**
   * 基于 K-Means 聚类与形态学后处理算法，在限定掩膜内全自动分离并计算真实的乳腺腺体实质体积。
5. **3D 安全边界距离热力图 (Safety Margin Heatmap)**
   * 提取结构表面网格，无缝计算内部腺体/病灶到最外层皮肤轮廓的空间 3D 最短距离，并渲染直观的伪彩热力图及数轴图例。

## 🏗️ 架构设计与计算优化

本插件已从早期的单文件脚本重构为专业的 **MVC 解耦架构**：
* **UI 表现层 (`Widget.py`)** 与 **核心算法层 (`Logic.py`)** 实现物理分离，便于后续在纯代码环境中进行批处理与算法扩展。
* **计算资源优化**：底层深度学习推理引擎针对单显卡（Single GPU）计算环境进行了严格的硬绑定与优化。这种设计有效规避了在非多卡集群环境下由于硬件轮询或框架初始化导致的进程锁死问题，确保了普通工作站上的极致稳定性。

## 🛠️ 依赖与环境要求 (Dependencies)

* **基础环境**：3D Slicer (建议版本 5.2 及以上)
* **算法依赖**：核心形态学与矩阵运算高度依赖 `numpy` 与 `scipy`。
* **深度学习与图像处理扩展**：在进行进阶的影像预处理或更换底层跨模态合成模型时，环境需配置 PyTorch、MONAI、SimpleITK 以及 OpenCV。

## 📥 安装指南 (Installation)

1. 下载或 `git clone` 本仓库到本地计算机。
2. 打开 3D Slicer，依次点击顶部菜单栏的 `Edit` -> `Application Settings`。
3. 在左侧面板选择 `Modules`。
4. 在右侧的 `Additional module paths` 列表中，点击 `+` 号（或 `Add` 按钮），将路径指向本仓库中**包含 `SimpleBreastDivider.py` 的根文件夹**。
5. 点击 `OK` 并重启 3D Slicer。
6. 重启后，在 Slicer 的模块选择下拉菜单中，进入 `Computational Medicine` 分类，即可找到并启动 **Simple Breast Divider**。

## 🚀 快速入门 (Quick Start)

1. **加载数据**：将患者的乳腺影像（如 MR 或 CT）拖入 3D Slicer 场景中。
2. **配置模型**：在模块顶部的“分割模型配置与执行”面板中，确认或修改乳房及病灶的 nnUNet 权重文件夹路径。
3. **执行分割**：依次点击执行乳房与病灶区域分割。系统会自动套用空间约束逻辑。
4. **进阶分析**：展开下方的各个折叠面板，根据临床需求执行“收缩计算”、“K-Means 提取”或“生成距离热力图”。
5. **可视化管理**：使用最底部的“3D 视图层级 management”面板，通过复选框与滑块实时调节各解剖结构的可见性与透明度。
